# 2026-06-16 main.py 动态蒙版改为动态背景建模

## 修改日期

2026-06-16

## 修改目标

不再使用“动态蒙版外挂过滤层”去持续屏蔽树叶、水面、反光等动态背景区域，而是把这些长期波动区域直接学习进背景模型本身。

目标是让系统后续仍然走“当前帧 vs 背景模型”的单一路径，只是背景容忍范围更宽，不再额外引入一层动态蒙版常驻过滤开销。

## 本次改动内容

### 1. 移除动态蒙版后处理思路

旧逻辑是：

1. 学习一段时间，得到一张“经常运动区域”的动态蒙版
2. 后续每轮帧差都再用这张动态蒙版去过滤运动 mask

新逻辑改为：

1. 学习一段时间的灰度样本
2. 对这段样本求中值背景
3. 同时统计该段视频下每个像素的波动范围
4. 生成更宽松的背景容忍阈值
5. 后续直接把它作为背景模型使用

### 2. 动态背景模型学习结果直接并入背景模型

动态背景学习完成后，会得到：

- `dynamic_bg_gray`
- `dynamic_bg_tol`

然后直接更新当前相机使用的：

- `bg_gray`
- `bg_tol`

后续帧差阶段仍然只调用：

- `apply_static_bg_mask(mask, gray, bg_gray, bg_tol)`

也就是说，运行时不再额外挂一层“动态蒙版过滤”。

### 3. 动态背景学习仍然保留串行调度

为了避免多路同时学习带来的板端压力，动态背景学习仍然保持逐路串行：

- Cam0 学完后，Cam1 再开始
- Cam1 学完后，Cam2 再开始
- 依次类推

## 参数变化

旧参数：

- `ENABLE_DYNAMIC_MOTION_MASK`
- `DYNAMIC_MOTION_MASK_*`

新参数：

- `ENABLE_DYNAMIC_BG_MODEL`
- `DYNAMIC_BG_SECONDS`
- `DYNAMIC_BG_SAMPLE_INTERVAL`
- `DYNAMIC_BG_START_DELAY_SECONDS`
- `DYNAMIC_BG_SERIAL_GAP_SECONDS`
- `DYNAMIC_BG_ABS_DELTA`
- `DYNAMIC_BG_STD_MULT`

## 影响范围

本次修改影响：

- 背景建模逻辑
- 动态背景抑制方式
- 帧差前背景过滤逻辑

本次未修改：

- 帧差生成主流程
- ROI 候选生成逻辑
- YOLO 推理方式
- 轨迹确认与绿框逻辑
- 数据发送协议

## 设计上的核心变化

这次的核心思想不是“额外记一张噪声黑名单图，后面每帧去扣它”，而是“把会晃动的树叶、水面、反光区域直接学成背景的一部分”。

因此后续运行时：

- 树叶晃动更容易被视为背景正常波动
- 真正外来目标仍然表现为超出背景容忍范围的变化

## 本地验证情况

本次为结构性改动，已完成代码级修改与文档记录，尚未完成新的板端实机验证。

## 后续验证重点

板端下一步重点观察：

1. 动态背景模型 ready 后是否仍会出现“学完一路就卡一路”
2. 树叶、反光、水面等动态背景是否能被更自然地吸收进背景模型
3. 对远距离无人机的响应度是否比旧动态蒙版方案更稳定
