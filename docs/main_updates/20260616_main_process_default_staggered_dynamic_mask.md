# 2026-06-16 main.py 多进程默认化与动态蒙版错峰启动

## 修改日期

2026-06-16

## 修改目标

本次修改主要处理板端启动阶段的运行稳定性问题，不改核心检测判定逻辑，重点是两件事：

1. 将多进程摄像头工作模式改成默认运行路径。
2. 将 5 路相机的 1 分钟动态蒙版学习改成按时间错峰启动，避免开局同时做动态统计导致启动拥塞。

## 本次改动内容

### 1. 多进程改为默认路径

主程序启动时不再依赖手动开关去决定是否启用多进程，而是优先尝试多进程运行。

当前行为是：

- 若运行环境支持 `fork`，默认每路相机一个独立 `Process`
- RKNN 推理仍由单独推理线程统一处理
- 若环境不支持 `fork`，则自动退回线程模式

### 2. 动态蒙版增加时序错峰

新增参数：

```python
DYNAMIC_MOTION_MASK_START_STAGGER_SECONDS = 8.0
```

含义如下：

- Cam0：立即开始动态蒙版学习
- Cam1：延后 8 秒
- Cam2：延后 16 秒
- Cam3：延后 24 秒
- Cam4：延后 32 秒

每路相机仍保持：

- `DYNAMIC_MOTION_MASK_SECONDS = 60.0`
- `DYNAMIC_MOTION_MASK_SAMPLE_INTERVAL = 0.5`

也就是说，学习窗口长度没变，只是各路不再同时开始。

### 3. 增加动态蒙版调试日志

新增了动态蒙版相关日志，便于板端观察：

- 什么时候开始排队等待学习
- 每一路相机什么时候真正开始学习
- 学习完成后的有效采样对数
- 最终动态蒙版屏蔽的像素数量

## 本地 Git 调用说明

这次暴露出一个流程问题：本地环境里虽然保留了 Git 的安装记录，但 `git` 实际不在当前可直接调用路径里，因此出现了“代码本地已改、说明已写，但没有及时完成 GitHub 推送”的问题。

后续处理规则：

1. 修改 `main.py` 后，必须先确认本地 `git` 是否能直接调用。
2. 如果 `git` 不在 `PATH` 中，必须先确认真实 `git.exe` 路径，再使用绝对路径执行。
3. 如果本地没有真实可用的 `git.exe`，则必须当场说明，不能默认视为已经具备推送条件。
