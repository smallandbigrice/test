# 2026-06-18 main.py 改为原始 ROI 推理

## 修改目标

让板端五路视频测试的 YOLO 输入与可正常检测的 `videomain.py` 保持一致，取消帧差特征与灰度图的三通道融合，直接使用原始画面裁剪出的 ROI。

## 本次修改

- 五路输入继续使用 `./4.mp4`，视频结束后自动从头播放。
- 保留静态背景蒙版、帧差、形态学处理和运动候选框提取。
- 删除一分钟动态背景学习及其逐路延时逻辑。
- 删除融合推理和融合显示开关及相关图像生成函数。
- 删除 tight padded canvas 推理路径。
- YOLO 输入统一改为 `frame[y1:y2, x1:x2].copy()`。
- 预览窗口统一显示原始画面，不再显示融合图。

## 算法影响

- 帧差：保留，仍用于寻找运动候选区域。
- 静态蒙版：保留，仍用于抑制固定背景。
- ROI：使用候选框在原始帧中直接裁剪，不填充到 640，也不叠加运动掩膜。
- YOLO：接收原始 ROI，由 `yololib.py` 按模型输入尺寸缩放。
- 轨迹与绿框确认：本次未修改。
- 启动等待：动态背景学习延时已删除；`INIT_TIME` 和静态背景采样等待仍保留。

- 已执行 `python -m py_compile main.py`，语法检查通过。
